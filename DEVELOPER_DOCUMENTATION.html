    Welcome to Posh Push

    Posh Push is a self-hosted, production-ready multi-tenant SaaS push notification platform. This documentation provides comprehensive guidance for developers integrating with, extending, or deploying Posh Push.

    What is Posh Push?

    A complete notification delivery system featuring:

    • ✅ Web Push + Firebase Cloud Messaging (FCM) delivery
    • ✅ Advanced subscriber segmentation with AI suggestions
    • ✅ A/B testing for notification optimization
    • ✅ Automation workflows (welcome, drip, scheduled sends)
    • ✅ Real-time analytics with CTR tracking
    • ✅ Multi-tenant architecture with role-based access
    • ✅ WordPress plugin for seamless integration
    • ✅ JavaScript SDK for custom implementations

    Core Concepts

    Sites (Tenants)

    Each user can manage multiple sites. Each site has:

    • Unique API key for SDK integration
    • Subscriber list
    • Notification settings
    • Analytics data

    Subscribers

    Users who have opted-in to receive notifications. Subscribers are:

    • Stored with unique identifiers
    • Grouped into segments
    • Targeted with notifications
    • Tracked for analytics

    Segments

    Groups of subscribers filtered by rules:

    • Behavioral (purchase history, activity)
    • Demographic (location, device type)
    • Custom (tags, attributes)
    • Dynamic (AI-generated suggestions)

    • Automations Module - Workflow automation

    • Admin Module - Platform administration
    • License Module - Plan enforcement
    • Webhooks Module - Event webhooks
    • A/B Test Module - Variant testing
    • Migration Module - Data import

    Request Flow Example

    Sending a Push Notification

    1. User clicks "Send Notification" in Frontend └─→ POST /api/notifications/send 2. Backend AuthModule validates JWT token └─→ Checks user permissions, site ownership 3. NotificationsController receives request └─→ Validates request data 4. NotificationsService processes logic └─→ Queries SegmentsModule for target subscribers └─→ Fetches subscriber list from database └─→ Creates notification record in database 5. AnalyticsModule begins tracking └─→ Initializes analytics entry 6. For each subscriber, send via web-push library └─→ Calls web-push.sendNotification() └─→ Browser Service Worker receives push └─→ Displays notification to user 7. User interaction tracked └─→ Click → Webhook to AnalyticsModule └─→ CTR updated in dashboard 8. Response returned to Frontend └─→ Show success toast notification

    JavaScript SDK Guide

    SDK Overview

    The Posh Push JavaScript SDK handles push notification subscription management and event handling on client websites.

    Installation

    CDN (Recommended for most use cases)

    <script src="https://your-domain.com/sdk/posh-push.min.js"></script> <script> PoshPush.init({ apiKey: 'your-site-api-key', serverUrl: 'https://your-domain.com' }); </script>

    NPM Package

    npm install posh-push import PoshPush from 'posh-push'; PoshPush.init({ apiKey: 'your-site-api-key', serverUrl: 'https://your-domain.com' });

    Basic Usage

    // Initialize with configuration PoshPush.init({ apiKey: 'your-api-key', // Required serverUrl: 'https://api.posh.com', // Required serviceWorkerPath: '/sw.js', // Optional promptDelay: 3000, // Optional: delay before showing prompt autoSubscribe: true, // Optional: auto-request permission debug: true // Optional: enable logging }); // Request notification permission PoshPush.requestPermission(); // Get subscription status const isSubscribed = await PoshPush.isSubscribed(); console.log('Subscribed:', isSubscribed); // Subscribe to notifications await PoshPush.subscribe(); // Unsubscribe await PoshPush.unsubscribe(); // Set user identifier (for tracking) PoshPush.setUserId('user@example.com'); // Add custom attributes PoshPush.setAttributes({ userName: 'John Doe', userPlan: 'premium', isVIP: true });

    Event Handlers

    // Handle notification received PoshPush.on('notification_received', (event) => { console.log('Notification received:', event.notification); }); // Handle notification clicked PoshPush.on('notification_clicked', (event) => { console.log('Notification clicked:', event.notification); // Redirect to URL if provided if (event.notification.data.url) { window.open(event.notification.data.url); } }); // Handle subscription changes PoshPush.on('subscription_changed', (event) => { console.log('Subscription changed:', event.isSubscribed); }); // Handle errors PoshPush.on('error', (error) => { console.error('SDK Error:', error); });

    AMP Integration

    For Google AMP pages, use the optimized AMP variant:

    <script async src="https://your-domain.com/sdk/posh-push-amp.js"></script> <script type="application/json" id="amp-posh-config"> { "apiKey": "your-api-key", "serverUrl": "https://your-domain.com" } </script>

    Blogger Integration

    For Blogger blogs, use the Blogger-specific variant:

    <script src="https://your-domain.com/sdk/posh-push-blogger.js"></script> <script> window.PoshPushConfig = { apiKey: 'your-api-key', serverUrl: 'https://your-domain.com', blogId: 'your-blogger-blog-id' }; </script>

    If you need a custom service worker, provide its path:

    Troubleshooting

    Common Issues

    Backend won't start

    Problem: Backend crashes on startup or won't connect to database
    # 1. Check database connection mysql -u root -p -h localhost # or psql -U postgres -h localhost # 2. Verify .env file is correct cat backend/.env # 3. Check database exists # MySQL: SHOW DATABASES; # PostgreSQL: \l # 4. Run migrations npm run db:migrate # 5. Check Node version node --version # Should be 18+ # 6. Clear node_modules and reinstall rm -rf node_modules backend/node_modules npm install # 7. View detailed logs NODE_ENV=development npm run backend:dev 2>&1 | tee debug.log

    Frontend won't load

    Problem: Frontend stuck on loading or shows blank page
    # 1. Check backend is running curl http://localhost:3000/health # 2. Verify API URL in .env cat frontend/.env # 3. Check browser console (F12) for errors # 4. Clear Next.js cache rm -rf frontend/.next # 5. Reinstall dependencies cd frontend npm install # 6. Start with verbose output npm run dev 2>&1

    SDK not working in browser
